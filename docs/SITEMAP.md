# Eliche Radiche Sitemap

**Lean • Focused • Customer-Centric**

A streamlined sitemap optimized for online presence, customer support, and live messaging.

---

## Primary Navigation

### 1. Home (`/`)
**Purpose**: Landing page, brand introduction, quick access to chat

**Key Elements**:
- Hero section with value proposition
- Key services overview (3-4 cards)
- Trust indicators (response time, availability)
- Live chat widget (always visible)
- CTA: "Request Service" button

**SEO Focus**: Brand keywords, location-based

---

### 2. Services (`/services`)
**Purpose**: Detailed service offerings

**Key Elements**:
- Service categories (Maintenance, Repairs, Inspections, Emergency)
- Service cards with descriptions
- Pricing indicators (starting from, or "Request Quote")
- FAQ section
- CTA: "Request Service" button

**Sub-pages**: None (keep it simple, all on one page)

---

### 3. Projects (`/projects`)
**Purpose**: Showcase completed work, build trust

**Key Elements**:
- Project gallery (before/after, yacht types)
- Filter by service type
- Brief case studies (3-5 featured)
- Testimonials integration
- CTA: "Start Your Project"

**Note**: Keep it visual, minimal text

---

### 4. About (`/about`)
**Purpose**: Build trust, humanize the brand

**Key Elements**:
- Company story (brief, premium tone)
- Team highlights (key technicians)
- Certifications & credentials
- Why choose us (3-4 points)
- CTA: "Get in Touch"

**Keep it**: Concise, professional, confident

---

### 5. Contact (`/contact`)
**Purpose**: Multiple contact methods

**Key Elements**:
- Contact form (name, email, phone, message)
- Contact information (phone, email, address)
- Office hours
- Map/location (if applicable)
- Live chat widget
- CTA: "Start Chat"

**Note**: Form submissions trigger email notification

---

### 6. Support (`/support`)
**Purpose**: Customer support hub, self-service

**Key Elements**:
- FAQ accordion (common questions)
- Live chat widget (prominent)
- Support resources (guides, tips)
- Emergency contact info
- Status page (if applicable)
- CTA: "Chat with Support"

**Focus**: Quick access to help, reduce support tickets

---

### 7. Request Service (`/request-service`)
**Purpose**: Service request flow

**Key Elements**:
- Step 1: Service type selection
- Step 2: Yacht details (type, size, location)
- Step 3: Urgency level
- Step 4: Contact information
- Step 5: Confirmation & next steps
- Live chat option throughout

**Post-submission**:
- Confirmation email
- Auto-create conversation in dashboard
- Operator notification
- Customer can track via chat widget

## Navigation Structure

```
Navbar:
├── Home
├── Services
├── Projects
├── About
├── Contact
└── [Chat Widget - Always Visible]

Footer:
├── Services (links)
├── Company (About, Contact)
├── Support (Support page, FAQ)
└── Legal (Privacy, Terms - if needed)
```

## User Flows

### Primary Flow: Service Request
1. Home → Services → Request Service
2. Or: Home → Request Service (direct)
3. Or: Chat Widget → Request Service

### Support Flow
1. Support page → FAQ (self-service)
2. Or: Support page → Chat Widget (live help)
3. Or: Any page → Chat Widget (always available)

### Discovery Flow
1. Home → Services → Projects → About → Contact
2. Or: Direct navigation via navbar
